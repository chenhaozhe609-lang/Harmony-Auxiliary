# Task 3: Piano Roll Editing, Playback Control, and Workspace Simplification

## 1. 背景

本轮反馈来自新的试用体验。Task 2 已经解决了节奏网格、长旋律、和声节奏、和声多轨显示和基础音色选择，但继续试用后暴露出三个更靠近真实工作流的问题：

- 手动输入虽然已经支持点击网格，但还不像 FL Studio 一类 DAW 的钢琴卷帘，音高定位和拖动编辑不够直觉。
- 播放控制存在明确 bug：点击暂停后，已经排队的音频仍会继续播放。
- 工作台按钮和设置过多，信息密度开始影响操作；落地页流程介绍文案也偏多，阅读引导不够强。

Task 3 的目标不是把产品变成完整 DAW，而是把当前 MVP 的输入、播放和界面组织提升到“更像音乐工具”的程度。

## 2. 产品目标

1. 手动输入区升级为更明确的 piano roll，音高、节奏、音符名称和网格完全对齐。
2. 播放控制符合用户预期：暂停能立即停止音频，从头播放和从当前小节播放有清晰入口。
3. 旋律音符支持横向和纵向拖动，横向改变时间，纵向改变音高。
4. 工作台控制项聚合收纳，降低第一屏按钮拥挤感。
5. 落地页流程介绍删掉冗余小字，让标题形成更清晰的单列阅读路径。

## 3. 非目标

本轮暂不做以下内容：

- 完整 DAW 时间轴、多轨录音或复杂 MIDI 编辑。
- 完整钢琴键盘视觉模拟，包括黑白键真实宽度。
- 采样级真实钢琴或吉他音源下载。
- MusicXML 导入导出。
- 撤销栈系统重构。

## 4. 任务拆分

### T3.1 播放暂停修复

目标：点击暂停后，音频必须真正停止，不再继续播放已排队的音符。

任务：

- 检查当前 `AudioEngine` 的调度方式。
- 避免一次性把所有未来音符直接排进 Tone timeline 后无法取消。
- 将播放调度改为可取消的本地 timer 或 Tone.Transport 可取消事件。
- 暂停时清理未来调度、释放当前发声，并保持 UI 状态一致。
- 保留已有 melody/harmony mute 行为。

验收标准：

- 点击播放后再点击暂停，旋律和和声音频立即停止。
- 暂停后不会继续响起后续音符。
- 播放头停止或重置行为与按钮语义一致。
- 单元测试或浏览器验证覆盖“播放后暂停不再继续触发音符”。

### T3.2 从头播放与从当前小节播放

目标：增加两个明确播放入口，让用户可以从全曲开头或当前小节开始听。

任务：

- 新增从头播放按钮。
- 新增从当前小节播放按钮。
- 根据当前 playback beat 或选中和弦位置计算当前小节起点。
- 播放时只调度起点之后的 melody 和 harmony。
- 播放头从指定 beat 起始，并继续按真实时间推进。
- 如果当前没有候选和声，按钮禁用。

验收标准：

- 从头播放始终从 beat 0 开始。
- 从当前小节播放会从当前播放头所在小节或选中和弦所在小节开始。
- 从中间播放时，之前的小节不会发声。
- 播放头位置与音频起点一致。

### T3.3 Piano Roll 输入区

目标：参考 FL Studio 的钢琴卷帘，把手动输入从“网格可点”升级为更清晰的旋律编辑器。

任务：

- 将 melody lane 视觉改为 piano roll 风格。
- 左侧固定显示 pitch row，例如 C5, B4, A4, G4 等。
- pitch label 与每一行旋律格子严格垂直对齐。
- 每个 pitch row 有清晰 hover / active 反馈。
- 点击某个 pitch row 和 beat cell 时创建音符。
- 保留当前 duration selector。
- 确保横向滚动时左侧音名保持 sticky。

验收标准：

- 用户能明确看出每一行对应哪个音高。
- 新建音符所在行与左侧音名完全对齐。
- 长旋律横向滚动时，左侧 pitch label 不丢失。
- 页面无横向整体溢出。

### T3.4 音符拖动改音高

目标：旋律音符可纵向拖动，直接改变 pitch。

任务：

- 扩展 note drag state，记录起始 X/Y 与原始 note。
- 横向拖动继续改变 `startBeat`。
- 纵向拖动根据 pitch row 改变 `midi`、`pitchClass`、`name`。
- resize 模式只改变 duration，不改变 pitch。
- 拖动结果吸附到当前半拍/八分格。
- 拖动后清空旧 harmony candidates，避免和声与新旋律不一致。

验收标准：

- 向上拖音符，音高升高并显示新音名。
- 向下拖音符，音高降低并显示新音名。
- 横向和纵向同时拖动时，时间与音高都正确更新。
- 生成和声使用拖动后的真实 pitch 和 startBeat。

### T3.5 工作台控制收纳

目标：减少工作台顶部和输入区域的按钮拥挤感。

任务：

- 将低频设置聚合到 settings/details 区域，例如 key、mode、tempo、rhythm、tone。
- 保留高频操作在主界面：播放、生成、导入/加载、输入模式。
- 将清空本地数据、恢复草稿、音轨选择等放入更局部的区域。
- 为 input actions 建立更明确分组：Import、Demo、Edit。
- 移动或合并重复的 Load Demo 入口。
- 确保移动端不会因为按钮过多换行挤压时间线。

验收标准：

- 第一屏按钮数量明显减少。
- 主操作路径更清楚：输入旋律 -> 生成和声 -> 播放/导出。
- 所有原有能力仍可访问。
- 桌面和移动端都没有按钮文字溢出。

### T3.6 落地页流程简化

目标：删掉流程介绍里的冗余小字，让标题形成单列阅读引导。

任务：

- 删除 workflow section 中每个标题下方的说明段落。
- 将流程标题调整为更一致的一列阅读结构。
- 保留必要视觉示例，但避免左右分散导致阅读跳跃。
- 保持第一屏品牌和 CTA 不变。
- 检查移动端标题换行和间距。

验收标准：

- 流程区不再出现标题下方长说明小字。
- 三个流程标题形成明确阅读顺序。
- 页面仍有视觉内容支撑，不变成纯文字列表。
- 移动端无文字重叠。

### T3.7 回归测试与浏览器验证

目标：避免播放、拖动和布局再次回归。

任务：

- 增加 AudioEngine 播放起点和停止行为测试。
- 增加 note drag pitch update 的 reducer 或 UI 验证。
- 增加 piano roll pitch label 对齐浏览器验证。
- 增加从当前小节播放的浏览器验证。
- 增加工作台按钮收纳后的 overflow 检查。
- 保留 Task 2 的长旋律验证脚本。

验收标准：

- `tsc -b` 通过。
- `vitest run --passWithNoTests` 通过。
- `vite build` 通过。
- 浏览器验证覆盖：
  - 手动创建音符。
  - 纵向拖动改音高。
  - 暂停后不继续播放。
  - 从头播放和从当前小节播放。
  - 工作台和落地页无横向溢出。

## 5. 建议实施顺序

1. T3.1 播放暂停修复。
2. T3.2 从头播放与从当前小节播放。
3. T3.4 音符拖动改音高。
4. T3.3 Piano Roll 输入区。
5. T3.5 工作台控制收纳。
6. T3.6 落地页流程简化。
7. T3.7 回归测试与浏览器验证。

原因：播放 bug 是明确体验问题，优先修；音符拖动和 piano roll 强相关，可以连续做；界面收纳和落地页简化最后统一调整，避免前面功能改动后又重复整理布局。

## 6. 风险与注意点

- 暂停修复要小心 Tone.js 调度方式，不能只调用 `releaseAll()`。
- 从中间播放时要确保和声和旋律使用同一个起始 beat。
- 纵向拖动要限制在现有 pitch rows 内，避免拖出可视范围后生成不可见音符。
- piano roll 视觉升级不能破坏已有长旋律横向滚动。
- 控制项收纳不能隐藏主路径操作，否则会让新用户更迷惑。
- 落地页删文案后仍要保留足够视觉节奏，避免信息过空。
